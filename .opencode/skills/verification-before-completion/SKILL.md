# Verification Before Completion

Final quality checks before marking work as complete.

## Quality Gates
- [ ] Task requirements fully met
- [ ] All tests pass
- [ ] No regressions
- [ ] Error paths handled
- [ ] Edge cases covered
- [ ] Performance impact acceptable
- [ ] Security reviewed
- [ ] Documentation updated
- [ ] No debug code remaining
- [ ] Changes are self-contained

## Verification Steps
1. Re-read the original requirements
2. Test the happy path end-to-end
3. Test error paths and edge cases
4. Check for regressions in related areas
5. Verify with actual user scenarios
