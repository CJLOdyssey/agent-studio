# Receiving Code Review

Responds gracefully to code review feedback.

## Process
1. Read all review comments thoroughly
2. Understand the concern behind each comment
3. Categorize: must-fix, should-fix, nice-to-have, discussion
4. Implement must-fix and should-fix changes
5. Respond to each comment
6. Thank reviewers for their input

## Response Guidelines
- **Agree**: "Fixed in abc123" with commit reference
- **Disagree**: Explain reasoning, suggest alternative
- **Unsure**: Ask clarifying questions
- Never take feedback personally
