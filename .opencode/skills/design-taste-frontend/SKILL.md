# Design Taste: Anti-AI-Slop Frontend

Ensures frontend code produces distinctive, tasteful designs that avoid the generic "AI slop" aesthetic.

## What to Avoid

- Purple gradients on white backgrounds
- Inter/Roboto/Arial as default fonts
- Generic card layouts with equal padding everywhere
- Overused color palettes (especially indigo/blue combos)
- Predictable hero sections (centered heading + description + CTA)

## What to Do Instead

- Commit to a BOLD, specific aesthetic direction before coding
- Use distinctive font pairings (display + body)
- Create intentional color systems with dominant/subordinate relationships
- Use unexpected layout compositions (asymmetry, overlap, grid-breaking)
- Add atmospheric details (textures, noise, subtle animations)
- Match complexity to vision (maximalist needs rich execution)

## Process

1. **Define Direction**: Ask user about purpose, audience, and desired feel
2. **Choose Aesthetic**: Pick from brutalism, editorial, retro-futuristic, organic, etc.
3. **Plan Visual System**: Typography, color, spacing, motion decisions
4. **Implement**: Write production code with attention to every detail
5. **Refine**: Review and polish until the vision is fully realized

## Key Principles

- Intentionality over intensity
- Distinctiveness over trend-following
- Cohesion across all visual elements
- Every choice should serve the aesthetic direction
