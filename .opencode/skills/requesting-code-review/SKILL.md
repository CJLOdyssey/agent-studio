# Requesting Code Review

Submits well-prepared code for review.

## Preparation
1. Review your own diff first
2. Write a clear PR title and description
3. Explain what was changed and why
4. Highlight areas of concern or uncertainty
5. Add screenshots for UI changes
6. Tag relevant reviewers

## PR Description Template
- **What**: Brief description of changes
- **Why**: Problem being solved
- **How**: Implementation approach
- **Testing**: How it was tested
- **Concerns**: Areas needing close review
