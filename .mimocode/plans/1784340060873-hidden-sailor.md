## 修复计划（按优先级排序）

### Phase 1: 修复现有失败测试（影响最大）

#### T1.1 标记后端集成测试
- **文件**: `tests/routers/auth/test_auth_api.py`
- **操作**: 给需要 Docker 栈的测试加 `@pytest.mark.integration` 标记
- **文件**: `tests/conftest.py` — 注册 `integration` marker
- **文件**: `pyproject.toml` — pytest 配置中 `addopts` 加 `-m "not integration"` 默认跳过
- **产出**: 单元测试不再报 10 FAIL + 3 ERROR

#### T1.2 排除 E2E 测试的 import 错误
- **问题**: `tests/e2e/` 下 9 个文件因 `from tests.conftest import ...` 失败
- **操作**: 检查 e2e conftest 是否独立，或修复 import 路径（加 `conftest.py` 到 `tests/e2e/`）
- **产出**: `--ignore=tests/e2e` 之外的 e2e 测试也能正常收集
