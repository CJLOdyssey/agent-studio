项目 7：虚拟软件外包团队 (Virtual Software Dev Team)

核心技能：AutoGen 或 CrewAI 框架、角色设定与通信协议。

项目描述：构建一个包含“产品经理”、“资深程序员”、“测试工程师”三个 Agent 的系统。输入一个简单的需求（如“写一个贪吃蛇游戏”），三个角色自动在群组中讨论，PM 输出文档，程序员写代码，测试员审查代码并提出修改意见。

挑战点：如何控制 Agent 之间的发言顺序？如何判断讨论已经收敛并结束对话（防止互相无限客套）？

目标：构建虚拟软件外包团队系统，输入需求，自动讨论并输出PM文档、程序员代码和测试审查意见
│
├─ [1,2] 主控制流程
│   ├─ [1,2] 配置管理
│   │   ├─ [0] 环境变量读取（如API密钥）
│   │   └─ [2] 生成框架配置对象（依赖AutoGen/CrewAI）
│   ├─ [1,2] Agent与群组构建
│   │   ├─ [2] 框架基础类导入（autogen/crewai）
│   │   ├─ [0] 角色系统提示模板（PM、程序员、测试员的行为描述）
│   │   ├─ [1,2] Agent实例化（组合提示与框架UserProxyAgent/AssistantAgent等）
│   │   └─ [2] 群组聊天创建（GroupChat、GroupChatManager）
│   ├─ [1,2] 对话执行与终止
│   │   ├─ [2] 框架对话启动器（initiate_chat）
│   │   ├─ [0] 用户需求输入处理
│   │   ├─ [0] 发言顺序控制（状态机逻辑）
│   │   ├─ [0] 收敛检测器（终止关键词/规则匹配）
│   │   └─ [0] 终止条件组合判断
│   ├─ [1] 输出提取与格式化
│   │   ├─ [0] PM文档提取器（正则匹配文档段落）
│   │   ├─ [0] 代码提取器（正则匹配代码块）
│   │   ├─ [0] 测试审查意见提取器（正则匹配意见段落）
│   │   └─ [1] 结果组装为JSON（依赖上述提取器）
│   └─ [0] 日志记录（logging标准库）
