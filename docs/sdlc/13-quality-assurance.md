## 8. 不符合项管理

### 8.1 不符合项分级

| 级别 | 定义 | 响应时限 | 升级路径 |
|------|------|---------|---------|
| 严重 | 影响项目目标达成或造成重大风险 | 24小时 | QA负责人→项目负责人→管理层 |
| 一般 | 局部偏离标准但不影响整体 | 5个工作日 | QA工程师→项目负责人 |
| 建议 | 可改进但不强制 | 下个迭代 | QA工程师→相关人员 |

### 8.2 不符合项处理流程

```
发现不符合项（评审/审计）
       ↓
    记录到不符合项跟踪表
       ↓
    分配责任人 + 整改期限
       ↓
    责任人实施整改
       ↓
    QA 验证整改效果
       ↓
   ┌─── 验证通过？───┐
   │                  │
   关闭             不通过 → 重新整改
```

### 11. 问题状态流转说明

质量问题从发现到关闭遵循以下状态流转规则：

```
[新建] → [已分配] → [处理中] → [已验证] → [已关闭]
   ↓          ↓          ↓
[已拒绝]  [重新打开]  [已挂起]
```

#### 状态定义

| 状态 | 定义 | 可转移状态 |
|------|------|------------|
| 新建 | 问题刚被记录，尚未分配责任人 | 已分配、已拒绝 |
| 已分配 | 问题已指派给具体责任人 | 处理中、已挂起 |
| 处理中 | 责任人正在整改问题 | 已验证、已挂起 |
| 已验证 | QA 已验证整改结果，确认有效 | 已关闭、重新打开 |
| 已关闭 | 问题已彻底解决，流程完结 | 重新打开（如后续发现未根治） |
| 已拒绝 | 经评估该问题不构成不符合项 | 已分配（如申诉成立） |
| 已挂起 | 因外部依赖等原因暂时无法处理 | 处理中、已分配 |
| 重新打开 | 已关闭的问题再次发生或验证未通过 | 处理中、已分配 |

#### 流转规则

1. **新建 → 已分配**：QA 负责人审核后指派责任人。
2. **新建 → 已拒绝**：经评估属误报或不构成不符合项，需记录拒绝理由。
3. **已分配 → 处理中**：责任人接受问题并开始整改。
4. **已分配 → 已挂起**：因资源、依赖等原因暂时无法处理，需记录挂起原因与预计恢复时间。
5. **处理中 → 已验证**：责任人完成整改并提交验证申请。
6. **处理中 → 已挂起**：整改过程中遇到阻塞，需记录原因。
7. **已验证 → 已关闭**：QA 验证通过，问题正式关闭。
8. **已验证 → 重新打开**：验证未通过，退回责任人重新处理。
9. **已关闭 → 重新打开**：问题复发或发现整改不彻底。
10. **已挂起 → 处理中 / 已分配**：阻塞解除，恢复处理。
