## Part B — 优化动因

### 2.1 现状

工作站（10 tab 的管理界面）CSS 分散在 8 个文件中，共 2360 行。10 个 tab 全量静态 import，使用 10 个 lucide-react 图标组件。

### 2.2 问题

1. **8 个 CSS import** → 8 个 HTTP 请求（开发）/ 8 个 chunk（生产）
2. **2360 行 CSS** → 大量重复、散装值（13 处 `border-radius: 6px`、5 处 `gap: 6px`）
3. **10 个 lucide-react 图标** → 增加 bundle 体积（tree-shaking 后仍影响）
4. **ErrorBoundary + i18n** → 非关键路径的 runtime 开销
5. **10 个静态 import** → 首屏加载所有 tab 组件

---

## Part C — 优化方案

### 3.1 设计 token 标准化

在 `tokens.css` 新增 16 个工作站专用 token：

```css
--ws-sidebar-width: 152px;
--ws-tab-gap: 6px;
--ws-tab-radius: 8px;
--ws-radius-sm: 4px;
--ws-radius-md: 6px;
--ws-radius-lg: 8px;
--ws-table-cell-padding: var(--sp-2) var(--sp-2);
```

批量替换 8 个 CSS 文件中的散装值（13 处 `border-radius`、10 处 `gap`）。

### 3.2 CSS 合并 + 精简

| 指标 | 优化前 | 优化后 | 降幅 |
|------|--------|--------|------|
| CSS 文件数 | 8 | 1 | -88% |
| CSS 行数 | 2360 | 50 | -98% |
| CSS import 行 | 8 | 1 | -88% |

### 3.3 组件精简

| 项 | 优化前 | 优化后 | 收益 |
|----|--------|--------|------|
| 图标 | `lucide-react` × 10 | emoji | 省 bundle（~3KB gzip） |
| ErrorBoundary | `react-error-boundary` | 移除 | 省 6KB |
| i18n | `useTranslation` + `t()` × 10 | 纯中文 | 省 60KB runtime |
| tab 加载 | 静态 import × 10 | `lazy()` + `Suspense` | code-split |
| Class 名 | `agentstudio-workstation-*` | `ws-*` | 少 70% 字符 |

### 3.4 工具链

Ponytail ultra 模式驱动的精简 → frontend-design 注入设计细节（编辑式极简主义）：

- sidebar 152px（比 140px 多 12px 呼吸空间）
- emoji 固定 20px 宽对齐
- 选中态 `color-mix(accent 8%)` 背景跟随主题
- 表头 `border-bottom: 2px` + `letter-spacing: 0.8px`
- sidebar 右侧渐隐装饰线（`::after` linear-gradient）
