# AGENT 管理页面设计文档

> **状态**: 设计草案
> **目标用户**: 技术用户（会调试 system prompt、绑定工具、选择模型）
> **管理范围**: 全部 Agent（系统团队 Agent + 自定义任务 Agent）

## 1. 背景

本项目是一个"AgentStudio"系统，用户可以通过配置不同类型的 AI Agent 组建虚拟团队完成软件项目。后端基于 FastAPI + LangGraph，已具备完整的 Agent/Team 数据模型和协作流程。

当前前端 `/agents` 页面已有团队分组列表和 Agent 配置弹窗的基础骨架，但在以下方面需要增强：
- 配置编辑能力较浅（缺少语法高亮、变量提示等辅助）
- 技能/工具/MCP 绑定无实际配置 UI
- 缺少配置克隆、导入导出
- 模型选择和温度等高级参数未在前端暴露
- 缺少批量操作和版本管理

## 2. 整体布局

### 2.1 页面结构

```
┌────────────┬────────────────────────────────────────────┐
│  侧边栏     │              主工作区                       │
│            │                                             │
│ 我的团队     │  [全部] [系统团队] [自定义团队]  [+ 新建 Agent] │
│ ◉ 系统团队   │                                             │
│   ├ 🤖 工具 │  ┌────┐ ┌────┐ ┌────┐                      │
│   └ 🤖 技能 │  │ 🤖 │ │ 👤 │ │ 🤖 │   卡片/表格切换       │
│            │  │工具 │ │产品 │ │前端 │                      │
│ ◉ 开发团队   │  │Agent│ │经理 │ │开发 │                      │
│   ├ 👤 产品 │  └────┘ └────┘ └────┘                      │
│   ├ 👤 前端 │                                             │
│   └ 👤 后端 │  ─── 表格视图 ───                            │
│            │  名称 │ 角色 │ 模型 │ 状态    │ 操作          │
│ [+ 新建团队]│  ───────────────────────────────────        │
│            │  工具Agent│ sys │ gpt-4o │ ● 启用 │ ✏️ 📋 🗑️  │
│ Agent 管理  │  产品经理 │ pm  │ ─      │ ● 启用 │ ✏️ 📋 🗑️  │
│ Skills     │  前端开发 │ fe  │ ─      │ ○ 停用 │ ✏️ 📋 🗑️  │
│ Tools      │                                             │
│ 系统设置    │                                             │
│ API 管理    │                                             │
└────────────┴────────────────────────────────────────────┘
```

### 2.2 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 视图模式 | 卡片 + 表格双视图切换 | 卡片适合快速浏览，表格适合批量操作 |
| 分类筛选 | Tab 分类：全部/系统团队/自定义团队 | 管理范围内的全部 Agent 需要快速切换视角 |
| 行操作 | 直接内联操作按钮 | 技术用户需要高效批量操作 |
| 团队树 | 左侧保留现有团队侧边栏 | 保持与现有 UI 结构一致，降低迁移成本 |

## 3. Agent 配置编辑器

### 3.1 触发方式

- 点击 Agent 卡片/行上的编辑按钮 → 打开配置弹窗（Modal）
- 点击"新建 Agent" → 打开空白配置弹窗
- 支持从已有 Agent「另存为模板」

### 3.3 Tab 详情

#### Tab 1: 系统提示词

```

│  ┌─ 工具栏 ─────────────────────────────────┐  │
│  │ [插入变量 ▼] [插入模板 ▼]  字数: 342     │  │
│  └──────────────────────────────────────────┘  │
│                                                 │
│  ┌─ 代码编辑器 ───────────────────────────────┐  │
│  │ 你是一个产品经理，负责分析用户需求并...       │  │
│  │ 你的工作流程：                              │  │
│  │ 1. 与用户沟通明确需求                       │  │
│  │ 2. 输出产品需求文档                         │  │
│  │ 3. 协调开发团队工作                         │  │
│  │                                            │  │
│  │ 输出约束：                                  │  │

**功能要点：**
- 语法高亮（Markdown 格式）
- 字数/Token 统计
- 变量插入：`{requirement}` `{session_context}` `{user_input}`
- 模板快速填充（预设角色模板）
- 行号显示

#### Tab 2: 模型参数

```
模型: [gpt-4o          ▼]
温度: [══════●════════] 0.7
Max Tokens: [4096    ]
超时时间: [120      ] 秒

```

**功能要点：**
- 模型下拉选择（从 API 提供商动态获取）
- 温度滑条（0.0 - 1.0，步长 0.1）
- 数值输入框（Max Tokens、超时、重试次数）
- 从环境变量/Key Vault 读取可用模型列表

#### Tab 3: 工具绑定

```
已绑定的工具:
  ☑ 读取文件    - 读取项目文件内容
  ☑ 执行命令    - 在沙箱中执行命令

**功能要点：**
- 从已有工具库勾选绑定

- 每个工具有简要描述
- 「管理工具库」链接到 `/tools` 页面

## 4. 测试沙箱

### 4.1 位置

配置弹窗右侧分栏，内嵌显示。

```
┌─── 配置面板 ───────────┬─── 测试沙箱 ───────────────┐
│                        │                            │
│  [Tab 内容]            │  > 帮我分析这个需求...       │
│                        │  > 好的，作为产品经理...     │
│                        │    经过分析，我建议...       │
│                        │                            │
│                        │  ── 调试信息 ──             │
│                        │  Tokens: 342/4096          │
│                        │  耗时: 2.3s  模型: gpt-4o   │
│                        │                            │
│                        │  [输入测试消息...]  [发送]    │
│  [保存]  [另存为模板]   │                            │
└────────────────────────┴────────────────────────────┘
```

### 4.2 功能要点

| 功能 | 说明 |
|------|------|
| 流式输出 | SSE 实时显示回复，支持逐字渲染 |
| 停止生成 | 中途可中断 |
| 调试面板 | 显示 Token 消耗、响应时间、模型名称 |
| 上下文关联 | 自动使用当前配置的 System Prompt、Tools、MCP |
| 配置即生效 | 修改 prompt 或参数后，下一条测试消息即用新配置 |

### 4.3 独立测试页面（可选）

除了内嵌沙箱外，保留一个独立的全屏测试页面入口，用于：
- 长时间对话测试
- 对比多个 Agent 的回复
- 分享测试结果

## 5. 辅助功能

### 5.1 导入/导出

```
导出 Agent ↘
  ┌────────────────┐
  │ 📄 agent-pm.json  │  点击导出 → 下载 JSON 文件
  └────────────────┘

导入 Agent ↗
  [拖拽文件到此处]  或  [选择文件]
  → 预览配置内容 → 确认（解决冲突）→ 导入
```

**导出格式示例：**
```json
{
  "name": "产品经理",
  "role_identifier": "pm",
  "system_prompt": "你是一个产品经理...",
  "model": "gpt-4o",
  "temperature": 0.7,
  "tools": ["read_file", "execute_command"],
  "mcp": [{"name": "数据库", "url": "postgresql://..."}],
  "skills": ["code-review"],
  "is_approver": true,
  "icon": "👤"
}
```

### 5.2 克隆 Agent

- 列表行操作中的克隆按钮
- 一键复制全部配置，名称自动加 "(副本)" 后缀
- 克隆后立即进入编辑状态（可选）

### 5.3 版本历史

```
版本历史:
  v3  2026-06-18 15:30  - 更新为 GPT-4o 模型
  v2  2026-06-17 10:00  - 添加工具绑定
  v1  2026-06-16 09:00  - 初始创建

[对比当前] [回滚到此版本]
```

每次保存自动创建快照，支持：
- 版本列表展示（时间戳 + 变更摘要）
- Diff 对比（两版本间差异高亮）
- 一键回滚

### 5.4 批量操作

在表格视图下：
- 多选 Agent（复选框）
- 批量启用/停用
- 批量删除（二次确认）
- 批量导出

## 6. 技术实现建议

### 6.1 前端

| 功能 | 建议方案 |
|------|----------|
| UI 框架 | 保持现有 React 技术栈 |
| 代码编辑器 | CodeMirror 6（轻量）或 Monaco（功能全） |
| 配置状态管理 | Zustand 或 React Context |
| API 调用 | 保持现有 axios/React Query 模式 |
| 弹窗组件 | 保持现有 Modal 组件，增强为分栏布局 |

### 6.2 后端 API（已有）

现有 API 基本覆盖，无需新增路由：

| 接口 | 已有 | 说明 |
|------|------|------|
| `GET /api/agents` | ✅ | 列表 |
| `POST /api/agents` | ✅ | 创建 |
| `PUT /api/agents/{id}` | ✅ | 更新 |
| `DELETE /api/agents/{id}` | ✅ | 删除 |
| `GET /api/teams` | ✅ | 团队列表 |
| `POST /api/teams/{id}/members` | ✅ | 添加成员 |
| `DELETE /api/teams/{id}/members/{member_id}` | ✅ | 移除成员 |

### 6.3 可能的扩展 API

| 接口 | 必要性 |
|------|--------|
| `POST /api/agents/batch-toggle` | 批量启停 |
| `POST /api/agents/{id}/clone` | 克隆 Agent |
| `GET /api/agents/{id}/versions` | 版本历史 |
| `POST /api/agents/{id}/test` | 测试运行（SSE） |
| `POST /api/agents/import` | 导入配置 |
