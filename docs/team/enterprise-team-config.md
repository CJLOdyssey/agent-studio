# 企业传统开发团队配置

> 适用于 20~100 人研发团队，中等复杂度项目

---

## 一、组织架构

```
                    CTO / 技术总监
                          │
        ┌─────────────────┼──────────────────┐
        │                 │                  │
   前端团队            后端团队           平台团队
   (5-8人)            (8-15人)           (3-5人)
        │                 │                  │
   ┌────┼────┐      ┌────┼────┐       ┌────┼────┐
   │    │    │      │    │    │       │    │    │
   FE   FE   FE    BE   BE   BE     DevOps SRE  QA
```

---

## 二、角色配置

### 2.1 产品与设计（3~5 人）

| 角色 | 人数 | 职责 |
|------|:----:|------|
| **Product Manager (PM)** | 1 | 需求收集、PRD、优先级排序、验收 |
| **UI/UX 设计师** | 1~2 | 界面设计、交互流程、设计规范 |
| **业务分析 (BA)** | 0~1 | 需求拆解、用户故事编写 |
| **技术写作 (TW)** | 0~1 | 技术文档、API 文档维护 |

### 2.2 前端团队（5~8 人）

| 角色 | 人数 | 职责 |
|------|:----:|------|
| **Frontend Tech Lead** | 1 | 架构、代码评审、技术决策 |
| **Frontend Engineer** | 3~5 | 页面实现、组件开发、状态管理 |
| **FE QA** | 1 | 前端自动化测试、E2E 测试 |

### 2.3 后端团队（8~15 人）

| 角色 | 人数 | 职责 |
|------|:----:|------|
| **Backend Tech Lead** | 1 | 系统架构、API 设计、技术决策 |
| **Backend Engineer** | 5~10 | 业务逻辑实现、数据库、接口开发 |
| **DBA** | 1 | 数据库设计、SQL 审核、性能优化 |
| **BE QA** | 1~2 | 接口测试、集成测试、性能测试 |

### 2.4 平台与支撑（3~5 人）

| 角色 | 人数 | 职责 |
|------|:----:|------|
| **DevOps Engineer** | 1~2 | CI/CD 管线、容器化、部署策略 |
| **SRE** | 1 | 线上监控、告警、容量规划 |
| **安全工程师** | 1 | 安全审计、渗透测试、合规 |

### 2.5 管理（2~3 人）

| 角色 | 人数 | 职责 |
|------|:----:|------|
| **Tech Lead / 架构师** | 1 | 全局技术方向、ADR |
| **Engineering Manager** | 1 | 团队管理、绩效、招聘 |
| **Project Manager** | 1 | 进度控制、跨团队协调 |

## 三、开发流程

### 3.1 迭代周期

```
2 周一迭代
│
├── Day 1-2: 需求评审 + 任务拆解
├── Day 3-5: 开发（前后端并行）
├── Day 6-7: 测试 + 联调
├── Day 8:   Code Review + 修复
├── Day 9:   Staging 部署 + QA 验证
└── Day 10:  发布 + 复盘
```

### 3.2 人员配比

```
一个完整功能模块的典型人员配置：

  1 个 PM（需求）
  1 个 Tech Lead（架构）
  2 个 Backend Engineer（实现）
  2 个 Frontend Engineer（实现）
  1 个 QA Engineer（测试）
  1 个 DevOps（部署）
  ──────────────
  8 人 / 功能模块
```

### 3.3 沟通层级

```
PM ──→ Tech Lead ──→ BE / FE
 │                      │
 └── QA ────────────────┘
      │
      └── DevOps / SRE
```

## 五、适用场景

| 适合 | 不适合 |
|------|--------|
| 产品需求明确、变动少 | 快速验证期 / MVP 阶段 |
| 团队规模 > 15 人 | 3~5 人小团队 |
| 项目周期 > 6 个月 | 1~2 个月短期项目 |
| 对稳定性和质量要求高 | 对速度要求极高的初创期 |

## 六、角色产出物清单

| 角色 | 必须产出 | 可选产出 |
|------|---------|---------|
| PM | PRD、用户故事、验收标准 | 竞品分析、数据报表 |
| Tech Lead | ADR、系统设计图、API 契约 | 技术规范、编码规范 |
| BE | 接口文档、单元测试、迁移脚本 | 性能报告 |
| FE | 组件代码、Storybook、E2E 测试 | 性能预算 |
| QA | 测试用例、测试报告、Bug 单 | 自动化回归脚本 |
| DevOps | CI/CD 配置、监控告警、部署手册 | 灾备方案 |
